## Current Implemented Scope

The current implementation includes repository hygiene, a local demo product, and the Milestone 1 DuckDB contracts/database slice. Deterministic rules, projections, projection contribution/backtest metrics, OR-Tools-backed optimizer with brute-force fallback, custom optimizer weights, request-specific recommendation fingerprints, chip scoring simulation, manual team/market/league/score imports, optional read-only fantasy access, mocked OpenF1 session-context fetching, mocked Jolpica season-context fetching, metadata-only RSS/news ingestion, a FastF1 cache adapter, data health degradation, provider registry and agent guardrails, fake provider fallback, league analysis, and the multi-page UI with UI-state/accessibility coverage are present for demo workflows.

The full v1 spec is not yet complete. Official provider SDK calls, projection hardening beyond deterministic fixtures, complete UI-state polish, and release-grade QA/docs remain open work.
